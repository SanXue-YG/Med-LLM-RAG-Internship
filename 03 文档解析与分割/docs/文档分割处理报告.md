# 文档分割处理报告

> **状态：已完成** — 验证样本 + 全量处理均已通过（2026-05-27）

## 1. 概述

本阶段将第二阶段输出的 `oa_comm_slim.jsonl`（4,557,627 篇文献）按已验证的分割策略，生成适合向量化检索的文本块数据集。

### 1.1 分割策略（来自第二阶段）

| 条件 | 策略 | 参数 |
|------|------|------|
| token ≤ 512 | 不分割（单块） | 直接作为一个 chunk |
| token > 512 | 滑动窗口分割 | `chunk_size=400`, `overlap=80` |

- **检索单元**：title + abstract
- **Tokenizer**：`sentence-transformers/all-MiniLM-L6-v2`
- **分割工具**：`langchain-text-splitters.RecursiveCharacterTextSplitter`

### 1.2 输出字段结构

```json
{
  "chunk_id": "PMC12345678",        // 单块直接用 pmcid，多块用 pmcid_chunk{i}
  "text": "Title\n\nAbstract...",  // title + abstract 拼接
  "doc_id": "PMC12345678",          // 原文献 ID
  "chunk_index": 0,                 // 块在原文中的序号（0-based）
  "total_chunks": 1,                // 原文被分成的总块数
  "source_title": "Article Title",  // 原文标题，便于追溯
  "token_count": 342,               // 当前 chunk 的 token 数
  "strategy": "single"              // single 或 sliding_window
}
```

## 8. 结论与规划

### 8.1 结论

1. **分割策略验证通过**：第二阶段制定的策略（单块 86% / 多块 14%）在全量数据上完全一致

3. **数据就绪**：6,107,296 个 chunk 已准备好供后续向量化使用

### 8.2 后续使用规划

1. **向量化入库**：直接读取 `oa_comm_chunks.jsonl`，按 `text` 字段生成 embedding
2. **检索追溯**：通过 `doc_id` 和 `chunk_index` 可定位原文献和块位置
3. **元数据保留**：`source_title` 可用于检索结果展示
